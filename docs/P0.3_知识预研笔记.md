### 1.1 NVIDIA Isaac Sim 官方 Manipulation 示例

**置信度**: 0.85 (基于官方文档和社区经验)

#### 核心特性

| 特性 | 描述 | GENESIS 适用性 |
|------|------|---------------|
| **物理仿真** | PhysX 5.x 后端，支持刚体、柔体、流体 | ✅ 高度适用，满足矿石/零件物理交互 |
| **域随机化** | 材质、光照、物理参数随机化 | ✅ Sim-to-Real 关键，需在 P8 阶段启用 |
| **并行环境** | Isaac Gym 支持 1024+ 并行环境 | ✅ RL 训练必备，P4.5 阶段使用 |
| **URDF/USD 导入** | 原生支持机器人模型格式 | ✅ P2 阶段机器人建模核心 |
| **语义分割** | 内置 semantic segmentation 输出 | ✅ P3 感知系统直接使用 |

#### 关键 API 模式

```python
# Isaac Sim 典型初始化模式 (置信度: 0.90)
from omni.isaac.kit import SimulationApp
simulation_app = SimulationApp({"headless": False})

from omni.isaac.core import World
from omni.isaac.core.robots import Robot
from omni.isaac.core.utils.stage import add_reference_to_stage

world = World(stage_units_in_meters=1.0)
world.scene.add(Robot(prim_path="/World/Robot", ...))
world.reset()

while simulation_app.is_running():
    world.step(render=True)
```

#### Manipulation 示例要点

1. **Franka Panda 抓取示例**: 展示力控抓取、阻抗控制
2. **物体检测**: 使用 `omni.isaac.core.utils.stage.get_current_stage()` + semantic API
3. **传感器**: RGB-D 相机、力/力矩传感器配置
4. **控制器**: ArticulationController 接口统一控制关节

#### 风险点

- ⚠️ **硬件要求高**: RTX 3060+ 最低，推荐 RTX 4090/A100
- ⚠️ **学习曲线陡峭**: Omniverse API 复杂，调试困难
- ⚠️ **版本兼容性**: Isaac Sim 4.x 与旧版 API 差异大

### 1.3 LeRobot (HuggingFace 具身智能框架)

**置信度**: 0.90 (活跃开发的开源项目)

#### 项目概述

- **仓库**: `huggingface/lerobot`
- **定位**: 端到端具身智能训练框架
- **特点**: 预训练模型、数据集、训练流程一体化

#### 核心架构

```
LeRobot 架构层次:
├── 数据层: HuggingFace Datasets (PushToHub)
├── 模型层: ACT, Diffusion Policy, TD-MPC
├── 环境层: Gym/Gymnasium 兼容接口
├── 训练层: PyTorch + WandB 集成
└── 部署层: ONNX 导出 + 实时推理
```

#### 关键技术

| 技术 | 描述 | GENESIS 应用 |
|------|------|-------------|
| **ACT (Action Chunking Transformer)** | 预测动作序列，减少延迟 | ✅ P4.5 RL 训练可选策略 |
| **Diffusion Policy** | 扩散模型生成动作 | 🔵 探索性使用，计算开销大 |
| **数据集格式** | HDF5 + Episode 结构 | ✅ 统一数据记录格式 |
| **Sim-to-Real** | 域适应技术 | ✅ P10 阶段关键 |

#### 代码模式

```python
# LeRobot 典型训练流程 (置信度: 0.85)
from lerobot.common.datasets.lerobot_dataset import LeRobotDataset
from lerobot.common.policies.act.modeling_act import ACTPolicy

dataset = LeRobotDataset("lerobot/pusht")
policy = ACTPolicy(
    input_shapes=dataset.meta.observation_shapes,
    output_shapes=dataset.meta.action_shapes,
    chunk_length=100,
)

# 训练循环
for batch in dataloader:
    loss = policy.forward(batch)
    loss.backward()
    optimizer.step()
```

#### GENESIS 集成建议

- ✅ **采用 LeRobot 数据格式**: 作为 P7 数据记录标准
- ✅ **使用 ACT 策略**: 作为 P4.5 RL 训练起点
- ⚠️ **需适配 Isaac Sim**: LeRobot 原生支持 Gym，需封装 Isaac Sim 环境

### 1.4 robosuite (机器人操作 Benchmark)

**置信度**: 0.90 (广泛使用的学术框架)

#### 项目概述

- **仓库**: `ARISE-Initiative/robosuite`
- **定位**: 机器人操作研究平台
- **特点**: 模块化环境设计、多种任务、RL 基准

#### 核心模块

```
robosuite 架构:
├── environments/
│   ├── ManipulationEnv (基类)
│   ├── Lift, Stack, NutAssembly, ... (具体任务)
├── robots/
│   ├── Sawyer, Panda, IIWA, UR5e (机器人模型)
├── controllers/
│   ├── OSC (Operational Space Control)
│   ├── IK (Inverse Kinematics)
│   ├── Joint Velocity
└── sensors/
    ├── Camera, ForceSensor, TouchSensor
```

#### 关键设计模式

```python
# robosuite 环境接口 (置信度: 0.95)
import robosuite as suite

env = suite.make(
    env_name="Lift",
    robots="Panda",
    controller_configs=suite.load_controller_config("OSC_POSE"),
    has_renderer=True,
    has_offscreen_renderer=True,
    camera_heights=480,
    camera_widths=640,
)

obs = env.reset()
for _ in range(100):
    action = policy(obs)  # 7-DOF action
    obs, reward, done, info = env.step(action)
    env.render()
```

| 控制器 | 特点 | 适用场景 | GENESIS 应用 |

| **IK** | 逆运动学位置控制 | 点到点运动 | ✅ P4.2.1 IK 求解器参考 |
| **Joint Velocity** | 关节速度控制 | 简单任务 | 🔵 不推荐 |

#### GENESIS 借鉴点

### 1.5 SayCan / Code-as-Policies

**置信度**: 0.80 (研究论文，实现多样)

#### 论文概述

| 论文 | 发表 | 核心思想 |
|------|------|----------|
| **SayCan** | ICML 2023 | LLM + 价值函数选择技能 |
| **Code-as-Policies** | ICRA 2023 | LLM 生成可执行代码 |

#### SayCan 架构

```
SayCan 流程:
1. 用户目标: "Make a cup of coffee"
2. LLM 生成候选技能: ["grasp_cup", "pour_water", ...]
3. 价值函数评估每个技能的可行性
4. 选择最优技能执行
5. 循环直到目标完成
```

#### Code-as-Policies 架构

```python
# Code-as-Policies 示例 (置信度: 0.75)
prompt = """
You are a robot control system. Given a goal, write Python code to achieve it.

Available functions:
- move_to(position)
- grasp(object)
- place(object, position)
- open(container)
- close(container)

Goal: Put the apple in the drawer.
"""

# LLM 输出代码
generated_code = """
move_to("apple")
grasp("apple")
open("drawer")
place("apple", "drawer")
close("drawer")
"""

exec(generated_code)  # 执行代码
```

#### GENESIS 应用

| 方面 | SayCan | Code-as-Policies | GENESIS 选择 |
|------|--------|------------------|--------------|
| **规划方式** | 技能选择 | 代码生成 | ✅ 混合使用 |
| **可解释性** | 中等 | 高 | ✅ Code-as-Policies 更可解释 |
| **安全性** | 需约束 | 需沙箱 | ⚠️ 必须添加安全检查 |
| **实时性** | 较慢 | 较慢 | ⚠️ 需缓存 + 预计算 |

#### P6 阶段设计建议

```python
# GENESIS Brain L3 设计 (置信度: 0.80)
```

```
└─────────────────────────────────────────────────────────────┘
```

### 3.2 关键决策理由

#### D1: 选择 Isaac Sim 而非纯 MuJoCo

**置信度**: 0.90

**理由**:
1. **渲染质量**: Isaac Sim 的光线追踪渲染对 Sim-to-Real 至关重要
2. **域随机化**: 内置材质/光照随机化，减少 sim-to-real gap
3. **传感器**: RGB-D + Semantic Segmentation 原生支持
4. **并行训练**: Isaac Gym/Lab 支持 1024+ 环境并行

**代价**:
- 学习曲线陡峭
- 硬件要求高 (RTX 3060+)
- 调试困难

#### D2: 选择 MJCF 而非 URDF

**置信度**: 0.85

**理由**:
1. **简洁性**: MJCF 比 URDF 更紧凑，减少配置错误
2. **MuJoCo 原生**: 快速原型验证
3. **可转换**: `mjcf2urdf` 工具可转换为 URDF，再转 USD

**代价**:
- 需要学习 MJCF 语法
- Isaac Sim 需要 USD 格式 (需转换)

#### D3: 选择 ACT 而非 Diffusion Policy

**置信度**: 0.80

**理由**:
1. **推理速度**: ACT 单次前向推理，Diffusion 需要多步去噪
2. **预训练**: LeRobot 提供 ACT 预训练权重
3. **实时性**: 操作任务需要低延迟响应

**代价**:
- ACT 对长序列任务效果可能不如 Diffusion
- 需要验证在 GENESIS 任务上的效果
